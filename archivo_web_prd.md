# archiv*o Web PRD
> 웹은 탐색·정리·아이디어화 허브 — 앱과 연동되는 영감 아카이브 플랫폼

---

## Overview

archiv*o 웹은 **쌓인 영감을 탐색하고, 정리하고, 아이디어로 발전시키는 공간**이다.
빠른 포착은 앱(추후 개발)의 역할이고, 웹은 그 영감들을 큰 화면에서 깊이 들여다보는 데 최적화된다.

브라우저 익스텐션을 통해 작업 중 링크를 바로 저장하는 것이 웹의 유일한 저장 시나리오이며,
이 외의 영감 포착은 앱에서 이루어진다.

**웹의 핵심 역할**
- 쌓인 영감을 큰 화면에서 탐색하고 정리
- 컬렉션으로 묶어 아이디어로 발전시키는 공간
- 브라우저 작업 중 링크를 원클릭으로 저장 (익스텐션)

---

## 웹 vs 앱 역할 분리

| 구분 | 웹 | 앱 (추후) |
|------|-----|---------|
| 주요 역할 | **탐색, 정리, 아이디어화** | **빠른 포착, 미션 수행** |
| 저장 방식 | 링크 (익스텐션 경유) | 사진, 텍스트, 링크 |
| 사용 맥락 | 데스크탑에서 작업·리서치 중 | 외출 중, 일상의 순간 |
| 미션 | 조회만 가능 (수행은 앱 전용) | 미션 생성 및 수행 |
| 공통 | 계정 연동 — 저장된 영감 실시간 동기화 |

> 웹에서 직접 링크/텍스트를 입력 저장하는 기능은 익스텐션이 없는 초기 베타 단계에서만 보조적으로 제공한다.

---

## Design Concept — 살아있는 아카이브

> **"영감이 쌓일수록 색이 피어난다"**

archiv*o의 핵심 비주얼 은유: 영감이 없는 공간은 무채색이고, 저장될수록 색이 섞이며 아카이브가 살아난다.

### 핵심 비유

관찰 → 영감 발견 → 영감 결합의 흐름을 시각적으로 표현한다.
비어있는 폴더(컬렉션)는 무채색의 정적인 공간이고, 영감이 쌓일수록 고유한 그라데이션이 형성된다.
이 그라데이션은 해당 컬렉션의 "영감의 밀도와 색깔"을 나타낸다.

### 상태별 시각 표현

| 상태 | 시각 표현 |
|------|-----------|
| 0개 (빈 컬렉션) | 무채색 — 회색 계열, 텍스처 없음 |
| 1~3개 | 아주 옅은 색조가 번지기 시작 (단색 틴트) |
| 4~8개 | 두 색이 섞이며 소프트 그라데이션 형성 |
| 9개 이상 | 풍부한 멀티-컬러 그라데이션, 광택감 |

### 시각 언어

- **단위 오브젝트**: archiv*o의 `*` (애스터리스크) — 영감이 쌓이는 그릇이자 상징
- **색상 생성 방식**: 저장된 영감의 관심사 태그를 기반으로 컬러 팔레트 자동 생성  
  예) 비주얼 → Purple, 테크 → Blue, 컬처 → Orange
- **전환 방식**: 새 영감 저장 시 그라데이션이 부드럽게 스며드는 트랜지션
- **표면 재질**: 유리(Glass) 느낌 — 반투명, 굴절, 빛 반사 (레퍼런스: 3D 애스터리스크 컨셉)

### 적용 위치 (우선순위)

1. **컬렉션 썸네일** — 가장 먼저 적용. 그리드에서 각 컬렉션이 고유한 색을 가짐
2. **홈 배경** — 전체 아카이브 밀도에 따라 배경 그라데이션이 미묘하게 변화
3. **온보딩 모달** — 관심사 선택 시 선택한 항목에 맞는 컬러가 배경에 스며드는 연출
4. **빈 상태(Empty State)** — 무채색 `*` 오브젝트가 화면 중앙에 표시, "첫 영감을 저장하면 색이 피어나요" 문구

### 참고 레퍼런스 무드

- Soft blur gradient orbs (유기적인 색 번짐)
- 3D glass asterisk — 빛이 굴절되는 유리 재질의 `*`
- 무채색 → 컬러 전환의 드라마틱한 대비

## Target Users

- **Primary**: 브라우저로 리서치하는 디자이너/기획자 — 작업 중 레퍼런스를 바로 담고 싶은 사람
- **Secondary**: 앱으로 포착한 영감을 정리하고 발전시키려는 사람

## MVP Features

### 🔴 Must Have

#### 1. 회원가입 / 로그인
- 이메일 + 비밀번호
- 소셜 로그인 (Google)
- 앱과 동일 계정 연동 기반

#### 2. 아카이브 홈
- 저장된 영감 카드 형태로 표시
- 최신순 기본 정렬
- 링크 / 텍스트 혼합 표시

#### 3. 태그
- 태그별 필터링
- 태그 수정 (웹에서 정리 목적)

#### 4. 컬렉션 (폴더)
- 영감을 묶어서 관리
- 예: "브랜딩 레퍼런스", "컬러 인사이트"
- 아이디어로 발전시키는 단위

#### 5. 링크 저장 (베타 보조)
- URL 직접 입력 → OG 이미지 / 제목 자동 추출
- 저장 시 "나의 한 마디" 메모 첨부
- **요약 기능 제외** (느낌 보존이 핵심)
- 익스텐션 출시 후 이 기능은 축소 또는 제거 검토

### 🟡 Should Have

#### 6. 검색
- 메모, 태그, 링크 제목 기반 검색

#### 7. 브라우저 익스텐션
- 브라우저에서 원클릭으로 현재 페이지 저장
- 웹의 킬러 저장 시나리오 — 가능한 한 빨리 출시

#### 8. 미션 조회
- 앱에서 생성된 오늘의 미션 웹에서 확인 가능
- 미션 수행(완료 처리)은 앱 전용

#### 9. 앱 연동 배너
- 앱 출시 후 연동 안내 및 다운로드 유도

### 🟢 Nice to Have (V2)

#### 10. AI 영감 연결
- 저장된 영감들 사이 연결고리 제안
- "이것들 합치면 이런 아이디어 어때?" 형태

#### 11. 텍스트 영감 저장
- 짧은 텍스트 메모 저장
- 앱이 없는 환경에서의 보조 저장 수단

## 페이지 구조

```
/                   랜딩 페이지 (서비스 소개)
/login              로그인
/signup             회원가입
/home               아카이브 홈 (영감 전체 목록)
/archive/[id]       영감 상세
/collection         컬렉션 목록
/collection/[id]    컬렉션 상세
```

---

## 연동 구조

```
앱 (모바일)          →  영감 포착, 미션 수행
    ↕ 동기화 (공통 API + DB)
웹 (브라우저)        →  영감 탐색, 정리, 아이디어화
```

- 계정 기반 동기화
- 앱에서 저장한 영감 → 웹에서 큰 화면으로 탐색
- 웹에서 정리한 컬렉션/태그 → 앱에서도 반영
- 공통 DB 설계 필요 (웹/앱 동일 스키마)
