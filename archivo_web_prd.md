# archiv*o Web PRD

> 웹은 단순 아카이브가 아니라, **생각을 펼치고 발전시키는 작업 공간**이다.

---

## Overview

archiv*o 웹은 흩어진 영감을 저장하는 공간을 넘어,
**영감을 연결하고 구조화하여 하나의 아이디어로 발전시키는 작업 공간**이다.

모바일 앱이 순간적인 포착을 담당한다면,
웹은 그 영감들을 **큰 화면에서 탐색하고, 정리하고, 확장하는 역할**을 한다.

특히 웹에서는 영감 간의 관계를 시각적으로 펼치고,
하나의 주제를 중심으로 생각을 발전시키는 데 집중한다.

---

## 웹 vs 앱 역할 분리

| 구분    | 웹                    | 앱 (추후)           |
| ----- | -------------------- | ---------------- |
| 주요 역할 | **구조화, 연결, 아이디어 확장** | **빠른 포착, 미션 수행** |
| 저장 방식 | 링크 (익스텐션 기반)         | 사진, 텍스트, 링크      |
| 사용 맥락 | 데스크탑 작업 중            | 이동 중, 일상         |
| 핵심 경험 | 생각 펼치기 (구조화)         | 순간 기록            |
| 공통    | 계정 연동 — 실시간 동기화      |                  |

## Core Concept — 생각의 깊이가 보이는 아카이브

> **"영감은 저장되는 것이 아니라, 생각을 거치며 깊어진다."**

archiv*o는 영감의 양이 아니라,
**그 영감을 얼마나 해석하고 발전시켰는지**를 중요하게 본다.

각 영감은 사용자의 행동에 따라 점점 “깊어지며”,
그 깊이는 색의 농도로 표현된다.

---

## 🌱 영감 성장 구조

각 링크는 사용자의 개입에 따라 3단계로 성장한다.

| 단계 | 행동          | 의미      | 시각 상태 |
| -- | ----------- | ------- | ----- |
| 1  | 저장          | 관심의 시작  | 회색    |
| 2  | 한 줄 인사이트 작성 | 이해      | 옅은 색  |
| 3  | 아이디어 확장     | 해석 및 적용 | 진한 색  |

* 색상 = 생각의 깊이
* 단계가 올라갈수록 시각적으로 더 또렷해짐

---

## 🎨 시각 시스템 — 색으로 드러나는 사고 과정

### 링크 단위

* 초기: 회색 (아직 해석되지 않은 상태)
* 인사이트 작성 후: 옅은 컬러
* 아이디어 확장 후: 진한 컬러

### 폴더 단위

* 내부 링크들의 색이 혼합되어 그라데이션 형성
* 폴더를 통해 “어떤 생각들이 쌓였는지” 직관적으로 파악 가능

👉 결과적으로, 사용자는 자신의 사고 밀도를 시각적으로 확인할 수 있다.

---

## 🧠 핵심 경험 구조

### 1. 링크 저장

* OG 이미지 및 제목 자동 추출

---

### 2. 링크 상세

* AI: 핵심 내용 3줄 요약 제공 (보조 역할)
* 사용자: 한 줄 인사이트 작성

👉 AI는 시작을 돕고,
👉 핵심 가치는 사용자의 해석에 있음

---

### 3. 아이디어 확장

* 해당 영감을 기반으로 추가 아이디어 작성
* 다른 영감과 연결 가능

---

### 4. 폴더 (작업 단위)

> 폴더는 단순 정리 공간이 아니라,
> **하나의 주제를 발전시키는 작업 단위다.**

* 여러 영감을 모아 하나의 흐름으로 구성
* 링크 간 관계를 기반으로 구조화 (마인드맵 형태 확장 가능)
* 폴더 단위 인사이트 생성

---

## 🧩 사용자 흐름 (핵심)

```
[앱 or 익스텐션]
→ 영감 저장

[웹]
→ 탐색
→ 한 줄 인사이트 작성
→ 아이디어 확장
→ 폴더에서 구조화
→ 하나의 생각으로 발전
```
